# BeamDrop Brand Guidelines

## Brand Position

BeamDrop is a premium native utility for private device-to-device transfer. It
should feel fast, controlled, professional, and calm. The product promise is:
move content between trusted devices without making the cloud the default path.

BeamDrop is not a playful sharing app, social product, generic file manager, or
web dashboard. It should feel like a first-class system utility that belongs on
Android, iPhone, macOS, and Windows.

## Personality

- Fast: actions are direct, progress is visible, and screens avoid ceremony.
- Premium: typography, spacing, motion, and copy feel deliberate and restrained.
- Minimal: only the next useful action is emphasized.
- Native: platform controls, navigation, and permissions are respected.
- Private: trust, approval, and local transfer status are visible without fear
  messaging.
- Professional: suitable for personal and work devices.

## Voice

BeamDrop copy should be concise and specific:

- Prefer "Send to MacBook Pro" over "Start transfer workflow".
- Prefer "Local network blocked. Try Manual IP." over "Something went wrong".
- Prefer "Scan the QR code on your other device" over abstract pairing language.

Avoid:

- Cute metaphors.
- Exclamation-heavy success copy.
- Fear-based privacy language.
- Technical protocol terms in primary UI.
- Marketing slogans inside the app.

## Product Promises

BeamDrop may say:

- "Send locally without logging in."
- "Pair trusted devices with QR code."
- "Approve unknown senders before receiving."
- "Resume large transfers after interruption."

BeamDrop must not say:

- "Automatic iPhone clipboard sync in the background."
- "Works on every Wi-Fi network automatically."
- "Anyone nearby can send instantly."
- "Cloud backup" unless a real cloud backup feature exists.

## Visual Direction

The visual language should be quiet and precise:

- Use native surfaces and platform blur/material only where the OS expects it.
- Use a restrained color system with a clear accent, not a rainbow palette.
- Favor crisp dividers, grouped lists, sheets, and panels over nested cards.
- Use device and transfer cards only for repeated items.
- Avoid decorative blobs, mascot imagery, web hero layouts, and dashboard chrome.

## Brand Objects

Primary brand objects:

- Trusted device.
- Transfer.
- QR pairing code.
- Local network route.
- Receive approval.

These objects should be recognizable across platforms while rendered using each
platform's native components.

## Motion Personality

Motion should communicate speed and certainty:

- Short transitions.
- Native sheet and navigation motion.
- Progress motion that is smooth but not theatrical.
- Reduced motion support for every animated state.

Avoid bounce-heavy, playful, or game-like motion.

## Implementation Rule

When brand consistency conflicts with native platform convention, native platform
convention wins. BeamDrop should feel like BeamDrop because of its product model,
copy, trust states, spacing discipline, and interaction quality, not because each
platform is forced into an identical custom shell.
